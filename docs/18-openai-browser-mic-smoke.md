## First attempt: hosted browser permission denial

The Browserbase/Hermes browser environment denied direct microphone access. The app surfaced the correct visual failure state:

```text

```

Telemetry reached the OpenAI broker session boundary:

```json

  "fixtureId": "browser-mic",
  "events": [{ "stage": "websocket_open" }]

```

Browser console had no JavaScript errors.

This confirms the visual permission-denial path remains deaf-first and visible.

## Controlled browser-mic-path smoke with synthetic MediaStream

Because the hosted browser could not grant a physical microphone, the second pass injected a local `navigator.mediaDevices.getUserMedia` override that returned a generated `MediaStream` from the bundled `two-speaker-captions.pcm` fixture. This still exercised the app's BrowserMicrophone path:

```text
getUserMedia -> MediaStream -> BrowserMicrophonePcmSource -> AudioController -> OpenAiLiveSession.sendPcmChunk -> broker /openai/transcribe -> OpenAI realtime transcription
```

This is not evidence from a real physical microphone. It is evidence that the browser microphone code path can push live PCM chunks to OpenAI through the broker and render resulting captions.

Injected stream facts:

```text
sampleCount: 54708
source duration: 3.41925 seconds
fake mic events: getUserMedia_called, source_started, source_ended
```

## Actual Chrome real microphone smoke

Antonio then approved the next gate: actual Chrome/browser microphone cloud-audio testing.

Environment:

- Local Chrome launched with a temporary profile and Chrome DevTools Protocol.
- `--use-fake-ui-for-media-stream` was used to auto-grant the browser permission prompt.
- No fake media device flag was used.
- Device enumeration showed multiple physical inputs. The default MacBook microphone returned zero amplitude in this run, so the smoke selected `Antonio’s iPhone Microphone` via a temporary in-page `getUserMedia` wrapper that still called native browser `getUserMedia` with a physical device ID.
- Audio source was macOS `say` played into the physical iPhone microphone path.

Important label: this is real browser microphone capture through a physical input device, but the spoken phrase was generated by macOS TTS rather than Antonio speaking live.
